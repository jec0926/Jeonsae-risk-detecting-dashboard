# Portfolio Upgrade Direction

## Executive Decision

JeonSAFE는 현재 상태에서도 공모전 완성본으로 충분한 스토리가 있습니다. 다만 포트폴리오에서는 "분석을 했다"보다 "정책 의사결정에 연결되는 리스크 제품을 설계했다"는 메시지가 더 강합니다.

따라서 고도화 방향은 새로운 머신러닝 모델 추가보다 다음 세 가지에 집중하는 것이 좋습니다.

1. 실제 산출물 기반 Streamlit 대시보드 재현
2. 경보 신호의 사후검증 정량화
3. PPT를 공모전 발표용에서 포트폴리오/면접 설명용으로 재구성

## Recommended Product Story

기존 메시지:

> 전세사기 고위험 지역을 식별하기 위한 종합 리스크 평가 모델

포트폴리오용 메시지:

> 거래, 등기, 보증사고 데이터를 연결해 지역 단위 전세사기 위험을 조기에 탐지하고, 공공기관의 점검 우선순위 산정에 활용할 수 있는 B2G 리스크 모니터링 시스템

핵심은 "모델 성능"이 아니라 "데이터가 정책 실행으로 이어지는 구조"입니다.

## Feature Upgrade Priorities

### 1. Dashboard as Evidence

현재 추가한 `data/jeonsafe_dashboard.csv.gz`는 실제 산출물 기반 대시보드 데이터입니다. 이 파일을 사용하면 GitHub에서도 실제 결과 기반 화면을 보여줄 수 있습니다.

대시보드에서 강조할 기능:

- 지역 선택: 시도, 시군구, 법정동
- 최신 월 기준 종합점수
- 위험알림과 사고알림 분리
- 전세가율, 거래건수, 근저당, 임차권, 보증사고 추이
- 경보 상세와 사후검증 문구

추가하면 좋은 기능:

- 경보 발생 월 자동 하이라이트
- Warning/Critical 지역 Top 10 랭킹
- 경보 후 6개월 사고 증가율 자동 계산
- 지도 시각화는 선택사항. 데이터 위치 좌표가 없으면 무리하지 않는 편이 좋음.

### 2. Validation Upgrade

가장 설득력 있는 보완은 사후검증을 표준화하는 것입니다.

추천 지표:

- Warning/Critical 발생 후 3/6/12/24개월 사고 증가율
- 경보 지역과 비경보 지역의 사고 증가율 차이
- 경보 발생 지역 중 이후 사고 증가 지역 비율
- 사고 급증 지역 중 사전 경보가 있었던 지역 비율
- 평균 lead time

면접에서 이 부분을 이렇게 설명할 수 있습니다.

> 단순히 위험 점수를 만든 것이 아니라, 경보 이후 실제 임차권등기와 보증사고가 증가했는지 추적해 조기경보 신호의 유효성을 검증했습니다.

### 3. PPT Upgrade

현재 PPT는 경진대회 발표 흐름으로 좋지만, 포트폴리오에서는 다음 순서가 더 효과적입니다.

1. Problem
   - 전세사기는 사후 대응 중심
   - 지역 단위 선제 점검 체계 부족

2. Data Architecture
   - 거래, 등기, 보증사고 데이터를 월별 지역 단위로 통합
   - 데이터 출처와 연결 구조를 한 장으로 설명

3. Risk Engine
   - 구조적 위험과 발생경보 분리
   - Z-score, rolling average, peer rank 기반 경보 로직

4. Dashboard
   - 실제 Streamlit 화면 캡처
   - 지역 선택, 점수 카드, 지표 추이, 경보 상세

5. Validation
   - 경보 전후 사고 증가율
   - 사례 지역 시계열
   - 경보가 노이즈가 아니라 조기 신호였다는 근거

6. Business/Policy Impact
   - HUG 보증심사
   - 국토교통부 점검 우선순위
   - 표적 점검에 따른 비용/피해 절감 시나리오

7. My Contribution
   - PM
   - 데이터 모델링
   - 위험지표 설계
   - 대시보드/서비스 구조 설계

## What Not To Overdo

다음은 우선순위가 낮습니다.

- 무리한 XGBoost/LightGBM 추가
- 라벨 정의가 불명확한 상태에서 AUC 강조
- 좌표 데이터 없이 지도 시각화에 많은 시간 투입
- 원본 공공데이터 전체를 GitHub에 업로드

특히 지도학습 분류 모델은 label leakage 위험이 있습니다. 보증사고나 임차권등기는 target과 feature의 경계가 흐려질 수 있으므로, 본 프로젝트는 설명 가능한 조기경보 엔진으로 포지셔닝하는 편이 더 안전하고 강합니다.

## Recommended GitHub Pitch

> JeonSAFE는 전세사기 위험을 개별 거래 단위가 아닌 지역-월 단위로 모니터링하는 조기경보 시스템입니다. 국토교통부 실거래가, 대법원 등기정보, 부동산테크/HUG 보증사고 데이터를 통합해 구조적 위험과 발생경보를 분리하고, Warning/Critical 알림을 생성했습니다. Streamlit 대시보드를 통해 위험 지표 추이와 경보 상세, 사후검증 결과를 확인할 수 있도록 구성했습니다.
