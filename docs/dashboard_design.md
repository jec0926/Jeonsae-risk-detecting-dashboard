# Dashboard Design

Streamlit 데모 영상과 발표자료의 서비스 소개 흐름을 기준으로, 대시보드는 다음 사용자 여정을 따릅니다.

## User Flow

1. 지역 선택

시도, 시군구, 법정동을 선택합니다. 로컬 환경에 실제 결과 파일이 있으면 전국/시도/시군구/법정동 전체 결과를 읽고, GitHub 환경에서는 샘플 데이터로 동작합니다.

2. 종합 리스크 확인

최신 월 기준으로 다음 정보를 즉시 확인합니다.

- 위험알림: None, Warning, Critical
- 사고알림: None, Warning
- 종합점수
- 구조적위험총점
- 발생경보총점
- 사고발생

3. 위험 지표 추이 확인

전세가율, 거래건수, 근저당권, 임차권등기, 보증사고 등 핵심 지표를 시계열로 확인합니다.
선그래프에는 경보 발생 월을 다이아몬드 마커로 강조하고, 전세가율 그래프에는 70% 주의선과 85% 고위험선을 표시합니다. 지표별 최근값, 전월 변화, 24개월 평균 배수, 데이터 커버리지까지 함께 보여 단순 차트가 아니라 경보 판단 근거를 설명할 수 있도록 구성합니다.

4. 이상치 상세 리포트 확인

경보 발생 월의 요약설명과 사후검증 문구를 확인합니다. 이 영역은 정책 담당자 또는 보증심사 담당자가 왜 해당 지역을 점검해야 하는지 설명하는 부분입니다.

## Layout

- Header: 서비스명, 선택 지역, 최신 기준 월, 위험/사고 알림 badge
- KPI Cards: 종합점수, 구조적위험, 발생경보, 사고발생
- Tabs:
  - 종합 리포트
  - 위험 지표 추이
  - 위험지역 랭킹
  - 경보 상세
  - 데이터

## Design Principles

- 발표자료의 B2G 서비스 톤을 유지하기 위해 행정/리스크 모니터링 화면처럼 구성
- 빨강은 Critical, 노랑은 Warning, 회색은 None으로 의미 고정
- 지표와 경보 사유를 함께 보여 설명 가능성 강화
- 실제 원본 데이터가 없어도 GitHub에서 샘플 데모 실행 가능
- 법정동 단위에서 구조 지표가 부족할 수 있으므로, 기본 화면은 시군구 `(전체)` 집계를 우선 보여주고 필요 시 법정동으로 내려가도록 구성
