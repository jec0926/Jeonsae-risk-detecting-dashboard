# JeonSAFE: 전세사기 지역 리스크 조기경보 대시보드

공공데이터 기반으로 전세사기 위험 신호를 지역 단위에서 탐지하고, 실제 사고 발생규모와 선행 위험지표를 함께 보여주는 Streamlit 대시보드 PoC입니다.

이 프로젝트는 고려대학교 2025학년도 DAB 공모전 **“전세사기 고위험 지역을 식별하기 위한 종합 리스크 평가 모델 개발”** 산출물을 GitHub 포트폴리오용으로 재구성한 버전입니다.

- DAB 소개: https://biz.korea.ac.kr/activities/DTB.html
- 원본 Streamlit 프로토타입과 현재 GitHub 버전은 화면 구성이 일부 다릅니다.
- 현재 버전은 재현 가능한 실행, 지표 설명, 포트폴리오 가독성을 우선해 다시 구성했습니다.

## 1. 문제 정의

전세사기 위험은 개별 계약 단위에서만 판단하기 어렵습니다. 특정 지역에서 보증사고, 임차권등기, 근저당 설정, 전세가율 상승 같은 신호가 누적되면 임차인 피해 가능성이 커질 수 있습니다.

기존 접근은 주로 사고 발생 이후의 사후 대응에 가깝습니다. 이 프로젝트는 지역별 공공데이터를 결합해 다음 질문에 답하는 것을 목표로 했습니다.

- 어느 지역에서 전세사기 위험 신호가 먼저 나타나는가?
- 실제 사고 발생규모가 큰 지역은 어디인가?
- 사고 발생규모를 설명할 수 있는 선행 위험지표는 무엇인가?
- 임차인, 정책 담당자, 금융/보증기관이 우선 점검해야 할 지역을 어떻게 선별할 수 있는가?

## 2. 데이터 출처

프로젝트는 전월세 거래, 매매 거래, 등기, 보증사고 관련 공공데이터를 지역·월 단위로 결합하는 구조로 설계했습니다.

| 데이터 | 활용 목적 | 주요 지표 |
| --- | --- | --- |
| 국토교통부 전월세 실거래가 | 임대차 시장 흐름 파악 | 전세 보증금, 계약년월, 지역 |
| 국토교통부 매매 실거래가 | 매매가 대비 전세가 수준 계산 | 매매가격, 전세가율 |
| 등기정보광장 등기 데이터 | 권리관계 위험 신호 파악 | 근저당 설정, 임차권등기 |
| HUG 보증사고 데이터 | 실제 사고 발생규모 확인 | 보증사고 건수, 사고금액 |

GitHub 버전에는 실행 가능한 압축 결과 데이터와 샘플 데이터를 포함했습니다.

| File | Description |
| --- | --- |
| `data/jeonsafe_dashboard.csv.gz` | 대시보드 실행용 압축 결과 데이터 |
| `data/sample_risk_result.csv` | 구조 확인용 샘플 데이터 |
| `scripts/build_dashboard_dataset.py` | 원본 결과 파일에서 GitHub용 대시보드 데이터를 재생성하는 스크립트 |

API 키는 저장소에 포함하지 않습니다. `.env.example`을 `.env`로 복사한 뒤 로컬 환경에서만 입력하는 구조입니다.

## 3. 모델 구조

모델은 블랙박스 예측 모델보다 설명 가능한 지역 단위 조기경보 체계를 목표로 했습니다.

```text
공공데이터 수집
  -> 지역·월 단위 정제
  -> 실제 발생규모 지표 생성
  -> 선행 위험지표 생성
  -> 전월 대비 변화율 / 24개월 rolling 기준 / 동월 peer rank 계산
  -> Warning / Critical 경보 생성
  -> Streamlit 대시보드 시각화
```

핵심 로직은 다음과 같습니다.

- 월별 지역 지표를 시계열로 정렬
- 전월 대비 급증 여부 계산
- 24개월 평균 대비 이례적 상승 여부 계산
- 같은 월의 다른 지역 대비 상위 위험 분위 계산
- 거래량이 지나치게 적은 지역은 경보 산정에서 보수적으로 처리
- 구조적 위험 신호와 실제 사고 발생 신호를 분리해 종합점수 생성

관련 설계 문서는 `docs/model_design.md`, 대시보드 구성은 `docs/dashboard_design.md`에 정리했습니다.

## 4. 위험지표 정의

이 프로젝트의 핵심은 위험지표를 **실제 발생규모**와 **선행 위험지표**로 분리하는 것입니다.

| 구분 | 역할 | 주요 지표 |
| --- | --- | --- |
| 실제 발생규모 | 위험 분류의 기준이 되는 결과 지표 | 보증사고 건수, 임차권등기 건수, 사고발생 합계 |
| 선행 위험지표 | 발생규모를 예측하거나 설명하는 변수 | 전세가율, 거래건수, 근저당 설정 건수, 신규계약비율, 소형주택비중 |

즉, “전세가율이 높으니 위험하다”가 아니라, **보증사고와 임차권등기 발생규모가 큰 지역을 위험 기준으로 보고 전세가율, 거래건수, 근저당 등은 그 위험을 설명하는 선행 변수로 활용**합니다.

이 구조는 포트폴리오 관점에서 다음 장점이 있습니다.

- 위험 판단의 기준과 설명 변수를 혼동하지 않습니다.
- 실제 사고 발생규모를 기준으로 모델의 타당성을 검증할 수 있습니다.
- 정책/보증/임차인 관점에서 우선 점검 지역을 설명하기 쉽습니다.

## 5. 서비스 화면 컨셉

현재 GitHub 버전의 Streamlit 앱은 다음 흐름으로 구성됩니다.

1. `종합 리포트`: 선택 지역의 최신 위험 상태와 점수 추이
2. `위험 지표 추이`: 실제 발생규모와 선행 변수의 월별 변화
3. `위험지역 랭킹`: 최신 월 기준 상위 위험지역 비교
4. `경보 상세`: 경보 발생 월과 요약 설명
5. `데이터`: 모델 산출 결과 테이블 확인

원본 공모전 당시에는 별도의 Streamlit 프로토타입 영상과 발표자료를 통해 서비스 방향을 설명했습니다. 현재 GitHub 버전은 그 컨셉을 유지하되, 공개 포트폴리오에서 실행과 설명이 쉽도록 UI와 코드 구조를 다시 정리했습니다.

주요 기능은 다음과 같습니다.

- 시도, 시군구, 법정동 단위 지역 필터
- 최신 월 기준 위험 경보와 사고 경보 표시
- 종합위험점수, 구조위험점수, 발생경보점수 추이
- 보증사고, 임차권등기, 근저당, 전세가율 등 지표별 월별 시각화
- 최신 월 기준 위험지역 랭킹
- 경보 상세 테이블과 원천 결과 테이블
- HUG, 등기정보광장 Open API 연동을 위한 커넥터 구조

## 6. 한계와 개선 방향

현재 버전은 포트폴리오용 PoC이므로 다음 한계가 있습니다.

- 일부 원본 데이터 출처와 API 구조가 변경되어 최신 데이터 자동 적재는 추가 보완이 필요합니다.
- HUG 보증사고 API는 시도 단위, 등기정보광장 API는 시군구 단위로 제공되어 행정구역 단위 정합성 처리가 필요합니다.
- 현재 GitHub 버전은 기존 산출 결과를 기반으로 실행되며, 실시간 운영 대시보드는 아닙니다.
- 위험 점수는 설명 가능한 규칙 기반 조기경보 구조이며, 운영 환경에서는 최신 사고 데이터로 재검증이 필요합니다.

향후 개선 방향은 다음과 같습니다.

- Open API 키 발급 후 월별 데이터 자동 수집 파이프라인 구성
- 보증사고와 임차권등기 발생규모를 라벨로 한 위험분류 모델 재학습
- 변수 중요도, SHAP, peer rank 등을 활용한 설명 가능성 강화
- 지도 기반 지역 시각화 추가
- 원본 Streamlit 프로토타입의 서비스 흐름을 현재 대시보드 UI와 통합

## 7. Open API 연동 계획

API 연동은 실제 키를 저장소에 넣지 않고, 커넥터 구조만 포함했습니다.

| Source | Purpose | Unit |
| --- | --- | --- |
| HUG 분양보증사고현황 API | 보증사고 발생규모 보강 | 시도 |
| 등기정보광장 근저당 설정 API | 선행 위험변수 보강 | 시군구 |
| 등기정보광장 임차권등기 설정 API | 실제 발생규모 보강 | 시군구 |

관련 정리는 `docs/api_sources.md`, API 테스트 메모는 `docs/api_test_notes.md`에 있습니다.

## 9. 실행 방법

```bash
git clone https://github.com/jec0926/Jeonsae-risk-detecting-dashboard.git
cd Jeonsae-risk-detecting-dashboard
pip install -r requirements.txt
streamlit run app/streamlit_app.py
```

브라우저에서 `http://localhost:8501`로 접속합니다.

원본 결과 파일에서 대시보드 데이터를 다시 만들려면 다음 명령을 실행합니다.

```bash
python scripts/build_dashboard_dataset.py
```
